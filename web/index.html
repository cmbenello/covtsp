The Problem

Covering TSP on a Time-Expanded Graph

Visit every station on a transit network as fast as possible. Unlike classical TSP, this is a Covering variant: you must visit a required set of nodes (not all), and you don't return to start.

The graph is time-dependent: you can only take a train when it departs. This means the problem is modeled on a time-expanded graph where each node is a (station, time) pair.

Edge types: transit arcs (ride a train), waiting arcs (wait for the next departure), and walking transfers (walk between nearby stations). 

Time-Expanded Graph

Station A ──(transit)──▸ Station B
  t=08:00                 t=08:04
     │                       │
  (wait)                  (wait)
     │                       │
  t=08:10                 t=08:15
     │                       │
     └──────(transit)──────▸ │
              t=08:10→08:14

   ╌╌╌ Walking Transfer ╌╌╌
Station A ~~(walk 3min)~~▸ Station C
  t=08:04                  t=08:07

Nodes: (station, departure_time)
Edges: transit | wait | walk
Weight: elapsed seconds

Methodology

Three-Layer Algorithm Stack

01

Problem Formulation

Covering TSP on a time-expanded directed graph built from GTFS timetable data. ~500K nodes for London. Decision variables: sequence of arcs minimizing total elapsed time.

02

Heuristic Solver

Greedy nearest-unvisited with beam search lookahead, followed by local search (2-opt, Or-opt, 3-opt) on the station visit order. Re-simulates the full timed path after each move.

03

LP Relaxation Bound

Flow-based LP relaxation provides a provable lower bound. The optimality gap = (heuristic - LP bound) / LP bound tells you how close to optimal the solution is. No algorithm can do better than the bound.
