How It Works

Algorithm Pipeline

Time-Expanded Graph

Details

Technical Details

Time-Expanded Graph

Unlike classical TSP, transit networks are time-dependent. Each node is a (station, departure_time) pair. Edges represent train legs, waiting arcs, and walking transfers.

Covering TSP

Visit a required set of nodes without returning to start. NP-hard in general. We use heuristics and compute LP relaxation bounds to prove solution quality.

GTFS Pipeline

Parses any city's General Transit Feed Specification data. Handles calendar filtering, service patterns, and computes walking transfers via KDTree spatial index.

LP Relaxation

Relaxing integer constraints gives a valid lower bound on the optimal solution. The integrality gap proves solution quality without exhaustive search.
