The Problem

Covering TSP on a Time-Expanded Graph

Visit every station on a transit network as fast as possible. Unlike classical TSP, this is a Covering variant: visit a required set of nodes, don't return to start.

The graph is time-dependent: you can only take a train when it departs. Modeled on a time-expanded graph where each node is a (station, time) pair. Edge types: transit arcs, waiting arcs, and walking transfers.

Methodology

Three-Layer Algorithm

01

Time-Expanded Graph

GTFS timetable data builds a directed graph with ~170K nodes for London. Dijkstra gives exact earliest-arrival queries.

02

Greedy + Local Search

Nearest-unvisited with beam search lookahead, improved by 2-opt and Or-opt moves on the station visit order.

03

LP Relaxation Bound

Flow-based LP provides a provable lower bound. The optimality gap tells you how close to optimal the solution is.
